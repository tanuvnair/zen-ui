# zen-ui 5.0.0

*2026-07-16*

One fix, to the pivot workbench. It is a major bump because the pivot looks and
behaves differently than it did yesterday — small, but you would notice it, and
that is the bar.

## The pivot's available fields now filter to one member, in React too

Drag a field into Rows or Values and its filter takes as many members as you
like — that is a real filter on real data. A field still sitting in **Available
Fields** has not been placed yet, so its filter is a preview: it answers *what
is in here*, and takes one member at a time.

The Solid binding always did this. React did not: its available fields
multi-selected, and the two bindings disagreed about what the same component
does.

**What changes for you**, if you use `PivotWorkbench` in React:

- Selecting a member in Available Fields now replaces the previous one instead
  of adding to it. Placed fields (Rows, Columns, Values) are untouched — they
  multi-select exactly as before.
- The indicator in that menu is now a **radio**, not a checkbox. It was drawing
  a square box, which is a promise you can tick more than one; you could, and
  that was the bug.

If you were relying on multi-select in Available Fields, move the field into
Rows or Values first — that is the zone that means "filter this for real".

Nothing else changed. No prop was renamed or removed, and the rest of the pivot
— drag, drop, keyboard, the 2D-windowed grid — is exactly as it was.

## Why this was worth a major

The prop existed the whole time. `singleSelect` was implemented in React's field
chip and its filter menu, and the chip forwarded it correctly. The workbench
simply never passed it: the object it builds for each chip set every property
except that one. Nothing was missing, nothing errored, and both bindings
type-checked — they just did different things, and only clicking them told you
so.
