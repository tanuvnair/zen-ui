# zen-ui 7.3.0

*2026-07-19*

Additive. Nothing you call changed and nothing looks different — this release
lets the Solid binding render on a server.

## `@algorisys/zen-ui-solid` now renders on the server

Until now, importing a zen-ui Solid component into a SolidStart (or any
Vinxi/Node) server threw at render time: the published bundle only carried
Solid's browser runtime, so `renderToString` had nothing to call. Server-side
rendering a page that used a zen-ui component meant not using it.

This release ships a second, server build alongside the browser one. Your server
renders the components to HTML and the browser hydrates them — with no change to
how you import or use anything:

```tsx
// app.tsx — unchanged
import { Button, Dialog, DataTable } from "@algorisys/zen-ui-solid";
```

Node picks the server build automatically through a `node` export condition; the
browser keeps loading the same DOM build it always has. Both bundles keep
`solid-js` external, so the server and client share one Solid instance and
hydration lines up. There is nothing to configure and no second import path.

If you render zen-ui Solid components purely on the client, this changes nothing
for you.
