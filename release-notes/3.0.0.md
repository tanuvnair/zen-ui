# zen-ui 3.0.0

*2026-07-15*

The first published release. `@algorisys/zen-ui-core`, `@algorisys/zen-ui-react`
and `@algorisys/zen-ui-solid` share one version, because they ship one API: a
component that exists in one binding and not the other is a bug here, not a
roadmap item.

## Breaking

**zen-ui no longer restyles the page around it.** It used to set your document's
root font size to 10px from its entry — a library reaching out and changing the
app it was dropped into. It now styles only the elements it renders, and the
element reset is opt-in:

```ts
import "@algorisys/zen-ui-react/preflight";
```

If you relied on the old reset, that import restores it. If you did not, you
have just stopped fighting it.

## What you get

Seventy-six components across both bindings, from `Button` to a drag-and-drop
`Pivot`. Rather than list them, the ones worth knowing about:

- **The application frame** — `ShellBar`, `DynamicPage`, `ObjectPageLayout`,
  `FlexibleColumnLayout`. The pieces a real screen is built from, not a page of
  primitives you assemble yourself.
- **The table ecosystem** — `DataTable`, `FilterBar`, `ValueHelp`,
  `SelectDialog`, `ViewSettingsDialog`, and a `Pivot` that windows in two
  dimensions. The *List Report* pattern demo shows them composed, which is the
  form most of them are actually used in.
- **`DynamicDateRange`** — date filters that stay current. Pick "Last 7 days" and
  the filter stores the *period*, not the two dates it happens to mean today, so
  a saved view still means the last seven days next month.
- **`Chart`** — line, area, bar, pie and donut. Every pie ships a hidden data
  table, because a shape carries no meaning into audio.
- **Theming** — override `--zen-*` custom properties; that is the whole public
  surface. Three built-in themes via `data-theme`.

## Notes worth reading before you upgrade

- **Spacing is not themeable.** `--zen-space-*` exists but the utilities do not
  read it: `.zen-p-4` compiles to a literal `1rem`, while
  `.zen-rounded-zen-md` compiles to `var(--zen-radius-md)`. Colour, radius and
  shadow are token-themeable; spacing is per-instance, via a prop or a class.
  See `/customizing` in either demo.
- **`Toast` is the one API that differs between bindings** — React wraps Radix
  Toast, Solid uses solid-toast. Converging it is open, and would mean picking a
  loser.
- **The Solid `Slider` takes `minValue`/`maxValue`** (Kobalte's vocabulary)
  where React takes `min`/`max` (Radix's).

## Fixed

The bugs worth naming, because each one shipped looking fine:

- **Dialogs and sheets were unreadable in dark mode** — they painted a dark
  panel and let the text inherit black from the page, at about 1.2:1. The token
  was right the whole time; nothing read it.
- **Rounded-corner overrides were silently ignored** in some combinations.
- **The pivot's drag deleted fields.** Dropping onto a zone that already held
  one resolved a field key as a zone name and removed the field instead of
  adding it — so every zone accepted exactly one field.
