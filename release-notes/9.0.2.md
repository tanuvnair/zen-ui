# 9.0.2

Right-to-left polish. If your app is left-to-right, nothing in this release
changes anything you can see.

## Buttons and icons inside components now mirror too

9.0.0 made zen-ui work right-to-left: layout mirrors, arrow keys follow the
reading direction, menus open the correct way. What it missed was the small
things positioned *inside* other things.

A dialog's close button sat in the top-right corner. In an RTL page, everything
around it flipped and it stayed exactly where it was — 13px from the right,
whichever way the page read. Same for the magnifier in a search field, the
show/hide button in a password field, the tick on a selectable card, the count
badge on a notification bell, and the checkmark beside a selected menu item.

All of them now sit at the **start** or **end** of the reading direction rather
than at a fixed side, so they move with the layout they belong to.

The padding that reserves room for them moved with them, which is the part that
would have bitten: a close button that mirrors while its header's padding does
not means the title slides underneath the button.

## Nothing to do

No API changed. In a left-to-right page the output is identical — `start`
resolves to left and `end` to right, so these are the same pixels they always
were. In a right-to-left page, things are where you would expect them for the
first time.

If you built a workaround — flipping an icon back with your own CSS, say — that
workaround is now the thing putting it on the wrong side.

## Still not mirrored, on purpose

`DataTable`'s column-resize grips stay on the physical right for now. They share
their maths with column pinning and sticky offsets, which compute positions
directly, so moving the grip alone would put it on the wrong edge of a pinned
column. That one needs doing as a piece with the pinning behaviour rather than
as a paint job.
