# zen-ui 6.0.0

*2026-07-16*

The pivot workbench looked like two different products depending on which
binding you used. It does not any more: Solid's now matches React's. If you use
`PivotWorkbench` in Solid, **your page will reflow** — that is the whole reason
this is a major.

## The Solid pivot workbench is laid out like React's now

Same props, same behaviour, same drop rules — the two bindings just drew them
differently, so a screenshot of one told you nothing about the other.

**What it looked like in Solid:** the `n rows · n cols` counter and **View Data**
were folded into the Available Fields header; **Values** and **Rows** were
stacked in a fixed 16rem sidebar down the left; **Columns** was a horizontal
strip across the top of the grid.

**What it looks like now**, as React always did: the counter and View Data sit in
their own bar along the top, Available Fields spans the width beneath it, and
**Values | Rows | Columns** are three equal columns under that — one per zone,
side by side, stacking to a single column on narrow screens.

**What changes for you**, if you use `PivotWorkbench` in Solid:

- **The workbench is a different height and shape.** If you sized a container
  around the old sidebar, or positioned anything against it, check it.
- **The grid area no longer has a hardcoded height.** Solid's pinned it to 500px
  on large screens and 350px below — your `children` had to live inside that
  whether or not it suited them. The area is now flexible, and your grid gets the
  space that is actually there. If you relied on the old fixed height, set it on
  your own grid.
- **No prop changed.** Nothing was renamed, removed or added. `fields`,
  `initialLayout`, `onLayoutApply`, `totalRows`, `totalCols`, `showBuilder`,
  `loadMembers`, `onClearFilters` all mean exactly what they meant.

React's workbench is unchanged in layout. Drag, drop, the keyboard path, Escape
to cancel and the 2D-windowed grid all work exactly as before in both — that is
pinned by a test both bindings run.

## Three things in Solid that were never about layout

Aligning the two surfaced real bugs sitting underneath the difference:

- **Row and column counts were formatted for one country.** Solid hardcoded
  Indian digit grouping, so `1234567` read as `12,34,567` regardless of who was
  looking — from a component nobody had told about a locale. It now formats to
  the reader's locale, as React already did.
- **"Clear filters" could appear with nothing to clear.** Solid counted a filter
  as active if the field had an entry at all, even when nothing was selected in
  it. It now asks whether the filter actually filters.
- **Available fields had a remove button that did nothing.** Clicking it moved
  the field to Available Fields, where it already was. It is gone — as it always
  was in React.

## And one in React

**The pivot's warning alerts had no icon.** "Value field required" and
"Dimension required" rendered an empty box where the icon belongs — the slot was
there, nothing was in it. Solid drew the icon correctly, so React was fixed to
match rather than the other way round.

## Why this was worth a major

Nothing broke, nothing errored, and both bindings type-checked and passed every
check in the repo the entire time. They simply drew the same component two ways,
and only looking at them side by side said so. A component library's output is
its API: if your page moves when you upgrade, that is a break, whatever the diff
looks like.
