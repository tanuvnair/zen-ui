# zen-ui 9.8.0

Nothing breaks. One addition, for the AI agents that build with zen-ui: the
skill now carries the real API, not just the catalogue.

## New: per-component API references in the agent skill

9.7.0 gave your coding agent a skill that says *which* component to use.
It still had to guess the props — and guessed props are how an agent writes
`activeStep` when the prop is `value`, or invents a variant that does not
exist, and the result fails silently as an unknown attribute.

The skill's `references/api/` folder now has one file per component family —
108 of them plus an index — listing the props that actually exist, extracted
from the source by the TypeScript compiler rather than written by hand:

- **zen-specific props in full**: name, type, and doc — including the variant
  unions (`variant?: "link" | "solid" | "outline" | "soft" | "ghost"`), which
  are exactly what gets hallucinated.
- **Item shapes** like `StepperStep` with their members — for a data-driven
  component, the item shape *is* the API.
- **Escape hatches** from the underlying primitives (`onEscapeKeyDown`,
  `forceMount`, …) by name, so the agent knows they exist without 290 inherited
  DOM attributes drowning them.

Because it is generated and checked in CI, a prop change cannot ship without
the reference updating — the reference is as current as the code, always.

If you installed the skill from 9.7.0, refresh your copy after upgrading:

```bash
cp -r node_modules/@algorisys/zen-ui-react/skills/zen-ui .claude/skills/
```
