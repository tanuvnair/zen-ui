## MessagePopover — the validation summary a long form owes its user

A form with twenty fields and three errors makes people hunt. `MessagePopover`
collects the messages into one button, counts them by severity, and — the part
that makes it worth having — takes the user to the field when they click one.

```jsx
import { MessagePopover } from "@algorisys/zen-ui-react";

<MessagePopover
  messages={[
    { id: "1", type: "error",   title: "VAT number is not valid",
      subtitle: "VAT number", targetId: "vat" },
    { id: "2", type: "warning", title: "Contact email is missing",
      subtitle: "Contact email", targetId: "contact" },
  ]}
/>
```

Give a message a `targetId` and clicking it closes the popover, scrolls that
element into view and **focuses** it. The focus matters as much as the scroll: a
`scrollIntoView` on its own moves the page but leaves a keyboard user exactly
where they were, which defeats the purpose. Elements that are not normally
focusable get a temporary `tabindex`, removed on blur, so your DOM is handed back
the way it was found.

### It does not read your form

You pass it messages; it renders them. It does not subscribe to a validation
library or decide when a field is invalid. That is deliberate — it works the same
with react-hook-form, with this package's `FormBuilder`, with modular-forms on
Solid, or with a `useState` of errors you built by hand.

### Two behaviours that are choices

**The trigger shows the worst severity present, not just a total.** "4 messages"
tells you nothing when one of them blocks submission, so the button carries the
icon of the highest severity in the list.

**The severity filter only appears when there is more than one kind.** A list of
four errors does not need a chip that says "Errors 4".

### Severity is the vocabulary you already have

`"error" | "warning" | "success" | "info"` — the same four words `Alert`,
`Banner` and `ObjectStatus` use. Not a fifth scale to learn.

### Options

- `onMessageSelect` — called on activation. It runs *alongside* the navigation,
  so you can open an accordion or switch a tab and still land on the field.
- `disableNavigation` — turn the focus/scroll behaviour off entirely.
- `emptyMessage`, `maxBodyHeight`, `triggerLabel`.

Available in all four bindings; the web-components one is
`<zen-message-popover>`, with `messages` set as a property rather than an
attribute since it is structured data.
