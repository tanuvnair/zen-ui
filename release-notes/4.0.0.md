# zen-ui 4.0.0

*2026-07-15*

A packaging release. No component changed, no prop changed, nothing looks
different — but zen-ui now ships about a tenth of the JavaScript it used to, and
TypeScript works for the first time.

## A `<Button>` was costing you 151 kB. It now costs 17 kB.

zen-ui was not tree-shakeable. Importing one component gave you the whole
library: a single `<Button>` cost **151 kB gzipped**, and importing nine
components on top of it added **330 bytes** — because you already had all of it.
Your bundler was doing its job; the package gave it nothing to work with.

Measured, gzipped, on a real consumer build:

| you import | 3.0.0 | 4.0.0 |
|---|---|---|
| one `<Button>` (React) | 151 kB | **17 kB** |
| nine components (React) | 151 kB | **57 kB** |
| one `<Button>` (Solid) | 83 kB | **16 kB** |

Import everything and you still pay for everything — that part is unchanged and
correct. What changed is that you now pay only for what you use.

**You do not need to do anything.** No import changes, no config. Rebuild and
your bundle shrinks.

The cause was two settings that only work together: the library was published as
one bundled module, and it never declared `sideEffects`. Bundled into one module,
`sideEffects` is all-or-nothing and the blob is used. Split into modules without
`sideEffects`, your bundler must assume every module does something on import and
keeps them all. Fixing either alone changes nothing — which is why this survived
so long.

## TypeScript now works

`package.json` pointed `types` at `dist/index.d.ts`. The build emitted
`dist/src/index.d.ts`. **Consumers got no types at all** — every zen-ui import
was silently `any`, with no error to tell you.

**This is why the major bump, and the one thing that can break you.** Code that
compiled against `any` now type-checks for real, so mistakes that were always
there may surface as new errors on upgrade. They are not new bugs; they are bugs
you could not see. If a build goes red, the errors are worth reading before
suppressing.

## Breaking

- **Types resolve now.** See above. Nothing else changes for a normal consumer.
- **`dist/` layout changed** — one `index.js` became one file per module. If you
  reached past the package entry into `@algorisys/zen-ui-react/dist/...`, that
  path is gone. Import from the package entry; deep paths into `dist/` were never
  a supported API.

## Under the hood

Three checks now guard this, because every part of it failed silently and
looked fine:

- `check:size` builds real consumer apps and weighs the gzipped output against a
  budget. A regression here is invisible to a build log.
- `check:package` asserts every path `package.json` promises actually exists on
  a clean `dist`. The broken `types` field survived a release because a stale
  file lingered on machines that had built the old layout once.
- `check:release` keeps the version consistent across the four places that
  describe a release.
