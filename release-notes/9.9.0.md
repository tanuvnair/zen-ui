# zen-ui 9.9.0

Nothing breaks. One new family, in all four bindings: media editing.

## New: MediaTimeline — a trim track for video UI

If you have ever built a "trim this video" screen, you have built this
component: a filmstrip with draggable range handles, a playhead, a timestamp
bubble under the cursor, and click-to-seek. Now it is one import.

```tsx
<MediaTimeline
  duration={video.duration}
  ranges={ranges}                 // [{ start, end }, …] — you own the array
  onRangesInput={setRanges}       // per pointermove, no history
  onRangesCommit={pushHistory}    // once, on release — wire undo here
  onSeek={(t) => (video.currentTime = t)}
  currentTime={playhead}
  thumbnails={frameUrls}          // your images; zen-ui never touches ffmpeg
/>
```

Three things worth knowing before you reach for it:

- **A range is just a range.** The component does not know whether a range
  means "cut this" or "keep this" — you decide, and `rangeClass` styles each
  range accordingly. The demo shows a red cut-segment editor built exactly
  this way.
- **The drag grammar is split for undo.** `onRangesInput` fires on every
  pointer move so the UI can follow live (and `onSeek` fires under the dragged
  edge so a preview can track it); `onRangesCommit` fires once on release with
  the final ranges. Commit is where history belongs.
- **It is controlled-only.** Like DataTable, the app owns the state: ranges,
  active range, zoom, playhead. There is no built-in zoom slider — `zoom` is a
  number you control, and a Slider next to the track is three lines of your
  own chrome.

Handles are keyboard-real: focus one, arrows nudge by the minimum step, Shift
steps a full second. Edge drags clamp against neighbouring ranges so ranges
cannot overlap, with the same math in every binding.

Not to be confused with `Timeline`, which lists events that happened. This one
edits time.

## New: Waveform — an audio lane with a trimmable clip

The other half of a Canva-style editor: audio as a lane under the video strip.
Feed it `peaks` — a plain `number[]` of amplitudes 0..1 that you decode once —
and it draws the envelope. Add a `clip` and you get a draggable window on the
lane: drag the body to place the audio (`offset`), drag the edges to trim it
(`start`/`end` within the audio). The left edge trims in place — the tail
stays put, the way every editor behaves.

```tsx
<Waveform
  peaks={peaks}
  duration={video.duration}      // the lane's time axis
  audioDuration={audio.duration} // the audio behind the peaks
  clip={clip}
  onClipInput={setClip}
  onClipCommit={pushHistory}
/>
```

Give a `Waveform` and a `MediaTimeline` the same `duration` and the same
`zoom` and the lanes align to the pixel — that is the whole mechanism for
stacking them into an editor surface.

## The details that carry across both

- **All four bindings**, same API: React, Solid, vanilla
  (`MediaTimeline({ … })` factories) and web components
  (`<zen-media-timeline>`, `<zen-waveform>`, with the drag grammar as
  `zen-ranges-input` / `zen-clip-commit` CustomEvents).
- `formatMediaTime(seconds)` is exported — the `HH:MM:SS.cc` formatter the
  tooltips use — and every timestamp display accepts your own via
  `formatTime`.
- The drag/clamp math lives in core and is contract-tested once for all four
  renderers, so a handle stops in exactly the same place everywhere.
