# 7.1.0

Three components you were building by hand, and the type and motion scales you
could not retheme. All additive — nothing you call changes.

## New: Search

A search field as a component, instead of the one you reassemble on every screen.

```jsx
<Search value={q} onValueChange={setQ} placeholder="Search components" />
```

Magnifier, a `type="search"` input (so the platform gives it `role="searchbox"`),
and a clear button that shows only when there is text and returns focus to the
field when you use it. `sm` / `md` / `lg`, controlled or uncontrolled. It exists
because zen-ui had built this exact thing seven times inside its own components —
ShellBar, ValueHelp, SelectDialog, DataTable, the select list, Combobox and
MultiCombobox — and none of the copies agreed.

## New: PasswordInput

A password field with a show/hide toggle.

```jsx
<PasswordInput placeholder="Password" autoComplete="current-password" />
```

The toggle is a real button — keyboard reachable, labelled, and its pressed state
is announced — not an icon only a mouse can hit. Flipping it never moves focus out
of the field. It wraps a native input, so `name`, `required`, `autoComplete` and
form submission all work unchanged.

## New: SkipToContent

The keyboard bypass a full app frame owes its users.

```jsx
<SkipToContent href="#main" />
{/* …ShellBar, Sidebar… */}
<main id="main" tabIndex={-1}>…</main>
```

It is the first focusable thing on the page and stays invisible until it takes
focus, so the first Tab reveals "Skip to main content" and Enter jumps past the
header and nav straight to the content. Now that zen-ui ships a whole app frame
(ShellBar + Sidebar + Page), a keyboard user should not have to tab through all of
it on every route. This is the WCAG 2.4.1 requirement, in about ten lines.

## You can now retheme type and motion

zen-ui's type scale and its animations used hardcoded values that no `--zen-*`
override could reach — a gap in the "override custom properties to retheme" promise.
That is closed. `--zen-font-*` (family, size scale, weights) and `--zen-duration-*`
/ `--zen-ease-*` now back every `zen-text-*`, `zen-font-*` and animation. Nothing
looks different — the defaults are the old values to the pixel — but you can now set
a brand font or retime the motion by overriding the tokens.

It also gives motion one honest answer to **reduced motion**. If a visitor's system
asks for less of it, the animations now shrink to nothing. They did not before,
because the timings were scattered across twelve keyframes with no shared place to
respond.

## Fixed

- **Solid: a standalone `<label for>` now associates with Checkbox, RadioGroup and
  Select.** The caller's `id` had been landing on a wrapper the browser cannot label,
  so clicking the label did nothing. It now lands on the native control, and the
  label both names and toggles it. (React and vanilla were never affected.)
